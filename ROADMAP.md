# Quant-Lab 学习路线图

> 本文档记录个人学习目标与阶段计划，与 README（对外展示）分开维护。
>
> 三个核心目标：
>
> 1. 提升工程实现与测试能力
> 2. 深入理解量化交易平台架构
> 3. 提升量化分析能力
>
> **实际用途：** 为个人基金持仓提供控仓决策支持（非实时，日度更新）。

---

## 阶段二：数据层（Data Layer）

### 目标：理解数据获取、清洗、缓存的工程模式

- [x] 定义 `DataFetcher` Protocol（接口抽象），理解为什么用 Protocol 而非继承
- [x] 实现 `AKShareAdapter`（A股）和 `YFinanceAdapter`（美股）两个适配器
- [x] 用 DuckDB 做本地缓存层，理解读写分离、缓存失效策略
- [x] 处理脏数据：复权、停牌、退市、涨跌停，写数据质量检查函数
- [x] 为数据层写单元测试（mock 外部 API 调用）

**学习点：** Protocol vs ABC、数据管道设计、SQL on local files

## 阶段三：因子工程（Factor Engineering）

### 目标：理解因子的本质与统计有效性

- [x] 实现基础因子：动量（1M/3M/12M）、波动率、换手率、价值类
- [x] 学习并实现 IC（Information Coefficient）分析
- [x] 计算 ICIR（IC / IC_std），理解它为什么衡量因子稳定性
- [x] 做因子分层回测（五分位数），画收益分布图
- [x] 实现多因子合成：等权、IC加权、PCA

**学习点：** 截面回归、因子暴露、多重共线性

---

## 阶段四：回测引擎（Backtesting Engine）

### 目标：理解回测的核心逻辑与常见陷阱

- [x] 实现向量化回测引擎（先理解 vectorized 的局限，再对比 event-driven）
- [x] 加入交易成本模型（双边手续费、冲击成本、持仓约束）
- [x] 理解并处理**前视偏差（look-ahead bias）**
- [x] 理解**生存者偏差（survivorship bias）**，思考如何在数据层规避
- [x] 处理调仓频率、再平衡逻辑

**学习点：** 两大回测偏差、向量化 vs 事件驱动的取舍

---

## 阶段五：风险管理（Risk & Attribution）

### 目标：能读懂并解释策略的风险来源

- [x] 实现完整风险指标：Sharpe / Sortino / Max Drawdown / Calmar / VaR / CVaR
- [x] 理解 Beta / Alpha（CAPM框架），实现回归计算
- [x] 实现 Brinson 归因（配置效应 vs 选股效应）
- [x] 画 Drawdown 曲线，分析水下时间分布
- [ ] 写一份策略的风险报告（文字 + 数据）

**学习点：** CAPM 框架、尾部风险度量、归因分析方法论

## 阶段六：策略开发（Strategy Development）

### 目标：从因子到可执行策略的全链路

- [x] **因子选股策略**：基于多因子打分的 Long-Only 组合
- [x] **配对交易**：找协整股票对，实现均值回归策略（理解协整 vs 相关性）
- [x] **ML Alpha**：用 LightGBM 预测截面收益，Walk-forward 验证（防止 data snooping）
- [x] 对比三种策略在不同市场环境下的表现

**学习点：** 协整检验（ADF / Johansen）、Walk-forward vs 简单训练集划分

## 阶段十：因子库扩充（Factor Library Expansion）

### 目标：构建更丰富的因子体系，提升选股信号质量

### 技术类因子（`quant/factor/`）

- [x] **RSI**：相对强弱指数，捕捉超买超卖信号
- [x] **MACD**：趋势跟踪因子，信号线与 MACD 线的差值
- [x] **布林带位置**：价格在布林带中的相对位置（均值回归信号）
- [x] **均线偏离度**：价格与 N 日均线的偏离百分比

### 统计类因子

- [x] **偏度 / 峰度**：收益分布的高阶矩，捕捉尾部风险偏好
- [x] **特质波动率**：去除市场 Beta 后的残差波动率

### 因子筛选

- [ ] 用 IC / ICIR 自动筛选有效因子，淘汰噪声因子
- [ ] 因子相关性矩阵，识别冗余因子

## 阶段十一：ML 模型库扩展（ML Model Library）

### 目标：对比多种模型，理解各自的归纳偏置与适用场景

- [x] **统一模型接口**：重构 `ml_alpha.py`，支持传入任意 sklearn 兼容模型
- [x] **Random Forest**：对比 LightGBM，理解 Bagging vs Boosting 的差异
- [x] **XGBoost**：与 LightGBM 横向对比，速度与精度的取舍
- [x] **Ridge / Lasso**：线性模型作为 baseline，理解正则化对因子权重的影响
- [x] **模型集成（Stacking）**：将多个模型预测合并，提升稳定性（依赖统一接口）

**学习点：** Bagging vs Boosting、正则化路径、模型集成方法论

## 阶段十二：全局配置系统（Config System）

### 目标：消除硬编码，让参数可追踪、可复现

> 现状问题：`TRAIN_DAYS = 240`、`TOP_N = 20`、`COMMISSION = 0.0003` 等魔术数字
> 散落在各脚本中，调参时无法追踪哪套参数产生了哪份结果。

- [x] 实现 `quant/config.py`：用 `dataclass` 或 `pydantic.BaseModel` 定义全局配置类
  - 回测参数：`train_window`、`predict_window`、`rebalance_freq`、`top_n`
  - 交易成本：`commission_rate`、`slippage`
  - 因子参数：各因子默认窗口（如 `rsi_window=14`、`momentum_windows=[20,60,120]`）
  - 数据参数：`data_dir`、`universe`、`start_date`、`end_date`
- [x] 支持从 YAML 文件加载，覆盖默认值（`configs/default.yaml`）
- [x] 将现有脚本中的硬编码逐步迁移到配置类
- [x] 配置对象随回测结果一起序列化保存，确保结果可复现

**学习点：** `dataclass` vs `pydantic`、配置即代码 vs 配置文件的取舍

---

## 阶段十三：因子参数优化（Factor Parameter Tuning）

### 目标：系统化地寻找稳健参数，而非凭经验猜测

> 注意：参数优化极易过拟合。本阶段的核心不是找到"最好"的参数，
> 而是理解如何识别真正稳健的参数区间。

- [ ] 对单因子做参数网格搜索（如动量窗口 10/20/40/60/120 天），记录各参数下的 IC/ICIR
- [ ] 画参数敏感性热力图：参数→IC 的响应曲面，寻找"平台区"而非"尖峰"
- [ ] IS/OOS 对比验证：在样本内最优参数是否在样本外依然有效
- [ ] 理解过拟合检测：参数数量 vs 数据量的比例，多重检验的 p-value 修正（Bonferroni）
- [ ] 将验证过的参数固化到配置系统（阶段十二）

**学习点：** 参数稳健性 vs 最优性、多重检验问题、样本内外分裂的正确姿势

---

## 阶段十四：市场环境判断（Market Regime Detection）

### 目标：识别大盘所处的市场状态，为仓位管理提供依据

> 这是基金控仓的核心模块。选股能力再强，若在熊市满仓也会亏钱。

- [x] 实现 `quant/regime/detector.py`：基于规则的市场状态分类（趋势 + 市场宽度 + 波动率比）
- [x] 输出三态：`BULL（趋势上行）/ RANGE（震荡）/ BEAR（趋势下行）`
- [x] 回测不同市场状态下各策略的表现差异（`scripts/backtest_regime.py`，BULL满仓/RANGE六成/BEAR两成）
- [x] 将 Regime 参数纳入全局配置（`RegimeConfig`）
- [ ] 在 Dashboard 增加市场状态面板

**学习点：** 技术分析的统计有效性、HMM 状态模型（可选进阶）

## 阶段十五：行业轮动分析（Sector Rotation）

### 目标：识别哪些行业/板块处于强势，辅助行业 ETF 配置

- [ ] 接入申万行业指数日度数据（`ak.sw_index_daily_indicator`）
- [ ] 计算各行业相对强弱（RS）：行业涨幅 / 沪深300涨幅的滚动比值
- [ ] 实现行业动量因子：过去 N 周行业 RS 的趋势
- [ ] 行业轮动可视化：热力图（行业 × 时间）、RS 雷达图
- [ ] 输出超配/低配建议：基于 RS 排名的行业权重建议
- [ ] 研究行业轮动与经济周期的关系（顺周期 vs 防御性行业）

**学习点：** 相对强弱、行业 ETF 选择逻辑、经济周期与行业映射

## 阶段十六：宏观因子接入（Macro Factors）

### 目标：将宏观经济指标纳入决策框架，提升对市场环境的解释力

> 对基金控仓而言，宏观因子往往比技术因子更能解释中长期趋势。

- [ ] 接入宏观数据（akshare 均有接口）：
  - **利率**：十年期国债收益率、银行间资金利率（DR007）
  - **经济活动**：PMI（制造业 + 服务业）、工业增加值
  - **流动性**：M1/M2 同比增速、社融规模
  - **通胀**：CPI、PPI
  - **外部**：美元指数、人民币汇率
- [ ] 计算各宏观指标与沪深300未来收益的相关性（滞后效应分析）
- [ ] 构建宏观景气度合成指标（类 PMI 合成法）
- [ ] 将宏观状态纳入市场环境判断（阶段十四）

**学习点：** 宏观-市场传导机制、滞后效应、多源数据融合

## 阶段十七：基金数据接入与评估（Fund Analysis）

### 目标：直接分析持有或候选的基金，而非只分析股票

- [x] 搭建数据接入框架（`quant/fund/loader.py`），支持 tushare `fund_nav` 接口
- [x] 实现基金评估指标：Sharpe、最大回撤、Calmar、近期动量、方向性建议（`scripts/fund_analysis.py`）
- [ ] ⚠️ 完整数据需要 tushare 高级权限（暂缓）：超额收益、风格分析、相关性矩阵
- [ ] 基金风格分析：将基金收益回归到风格因子（价值/成长/小盘/大盘），理解实际风险暴露
- [ ] 基金相关性矩阵：识别持仓中重叠度高的基金（避免伪分散）
- [ ] 基金筛选评分卡：综合业绩、风险、风格稳定性的量化评分

**学习点：** 基金业绩归因、风格漂移识别、组合层面的风险集中度

## 阶段十八：仓位建议引擎（Position Sizing Engine）

### 目标：综合所有信号，输出可操作的仓位建议

> 这是整个系统的"出口"——把前面所有分析汇聚成一个具体建议。

- [ ] 实现 `quant/advisor/position.py`：仓位建议模块
  - 输入：市场状态（阶段十四）+ 行业信号（阶段十五）+ 宏观景气（阶段十六）
  - 输出：建议总仓位（0%~100%）+ 各行业 ETF 权重建议
- [ ] 仓位计算方法：
  - **波动率目标法**：根据当前市场波动率，反推持仓比例，使组合预期波动率固定
  - **信号加权法**：各信号赋权后加总，映射到仓位区间
- [ ] 历史回测仓位建议的有效性（信号→仓位→收益的归因）
- [ ] Dashboard 增加"今日建议"页面，显示最新仓位建议和信号来源

**学习点：** Kelly 公式、波动率目标法、信号聚合与权重设计

## 推进顺序建议

```text
已完成：阶段一 → 二 → 三 → 四 → 五 → 六 → 七 → 十一 → 十二 → 十四（部分）→ 十七（框架）

近期优先（基础设施）：
  阶段十二（配置系统）← 最先做，后续所有阶段都依赖它
  阶段十一（ML统一接口）← 顺手重构，影响范围小

中期目标（控仓核心）：
  阶段十四（市场环境判断）
  → 阶段十五（行业轮动）
  → 阶段十六（宏观因子）
  → 阶段十八（仓位建议引擎）

穿插进行：
  阶段八（测试质量）← 每写一个新模块就补测试
  阶段十三（参数优化）← 配置系统完成后再做
  阶段十七（基金评估）← 控仓核心链路稳定后再加
  阶段十九（定期报告）← 最后收尾，把所有输出串联
```

每完成一个阶段，建议在 `docs/` 下写一篇简短的技术笔记，记录遇到的坑和决策理由。
